# Criteria – UI/UX (30 points)

- Your application should be easy to use, and comfortable to see.
  - This means, your points will be deducted when:
    Text is too hard to read, View is too small to touch, View is not properly aligned, View is overlapping strangely, Navigation(back button) is not working properly, …
  - cf) https://material.io/design

- Open source UI is allowed…,
  but you must write its original source(Github Link) in your report.

- When your level of UI/UX is similar to weekly lab session, you can get minimum 10 pts to this section.

# 추가 공지 정리

## 학기말 개인 프로젝트 세부 사항

- 본 프로젝트는 11주차 강의 시간에 추가 설명이 진행될 예정입니다.
- 아래 가이드라인과 PDF 내용을 함께 참고해야 합니다.

## 주제

- 자유 주제의 애플리케이션 구현
- Topic: Implementing an application of your choice (Free subject)

## 필수 조건

1. Activities
   - 애플리케이션 내에 3개 이상의 Activity가 포함되어야 합니다.
   - Your application must contain 3 or more Activities.

2. API 및 네트워킹
   - Open API와 통신하고 네트워크 연결이 되어야 합니다.
   - 직접 구축한 AWS, Firebase 등의 자체 서버와 통신하는 경우에도 조건 충족으로 인정됩니다.
   - Your application must communicate with an Open API and require network connectivity.
   - Communicating with your own custom-built server, such as AWS or Firebase, is also accepted.

3. 개발 언어
   - Kotlin을 사용하여 제작해야 합니다.
   - Your application must be developed using Kotlin.

## 세부 평가 기준

| 항목 | 비율 |
| --- | ---: |
| 구현 (Implementation) | 40% |
| 디자인 및 사용성 (UI/UX) | 30% |
| 영상 및 데모 발표 (Presentation & Demo) | 15% |
| 독창성 (Novelty) | 10% |
| 보고서 (Report) | 5% |

## 제출 일정

| 제출 항목 | 마감 |
| --- | --- |
| 발표 영상 및 보고서 | 6월 12일(수) 23:59까지 iCampus 제출 |
| 프로젝트 코드 (Zip 압축) | 본인 데모 시연일(6월 17일) 23:59까지 iCampus 제출 |
| 데모 시연 | 마지막 주 수업 시간에 본인 노트북 지참 후 직접 시연 |

- 프로젝트 코드 제출이 늦을 경우 10점 감점됩니다.
- 데모 시연은 조교 안내에 따라 진행되며, 발표 순서와 일정은 추후 공지됩니다.
- 지연 제출은 원칙적으로 허용되지 않습니다. 단, 코드 제출은 위 감점 규정이 적용됩니다.

## 보고서 및 영상 유의사항

- 보고서와 영상은 한국어 또는 영어 중 편한 언어를 사용할 수 있습니다.
- 영상 형식은 자유입니다.
  - 목소리 유무 자유
  - PPT 사용 여부 자유
  - 실행 화면 포함 여부 자유
- 영상 자체의 퀄리티는 평가하지 않습니다.
- 영상은 아래 조건을 충족해야 합니다.
  - 5분 이내
  - 앱의 주제 포함
  - 구현 과정의 challenge 포함
- 보고서는 5페이지 이내 요약 작성을 권장합니다.
- 보고서 분량 자체에 따른 추가 감점은 없습니다.
- 보고서는 PDF 형식으로 기한 내 제출해야 합니다.
- 보고서에는 반드시 아래 내용을 포함해야 기본 보고서 점수 5점을 받을 수 있습니다.
  - 학번
  - 이름
  - 앱 이름
- 보고서 내용이 불충분하여 앱을 파악하기 어려운 경우 UI/UX나 구현 채점에 불이익이 있을 수 있습니다.

## 서버 통신 난이도 관련

- Firebase 사용도 구현 조건으로 인정됩니다.
- 다만 AWS 등을 이용해 직접 서버를 구축한 경우보다 구현 난이도 면에서 낮게 평가될 가능성이 있습니다.
- 직접 서버를 구축한 경우, 프로젝트 코드 압축본에 반드시 아래 내용을 포함해야 합니다.
  - 서버 설정
  - 관련 서버 코드

## 평가 연계성

- 영상이나 보고서에 언급하지 않은 내용은 데모 시간에 추가로 확인하지 않습니다.
- 반대로 영상이나 보고서에 언급했지만 실제 데모에서 구현되지 않았거나 동작하지 않는 항목은 엄격히 채점에 반영됩니다.
- 따라서 발표/보고서에는 실제로 안정적으로 동작하는 기능 위주로 작성하는 것이 좋습니다.

## 부정행위 및 오픈소스 사용

- 웹사이트나 타인의 코드를 무단으로 도용하는 행위는 금지됩니다.
- 정당한 라이선스를 가진 오픈소스를 활용할 수 있습니다.
  - 예: MIT License
- 오픈소스를 사용할 경우 반드시 출처를 명시해야 합니다.
- 출처를 명시하지 않거나 부정행위가 확인될 경우 0점을 받을 수 있습니다.

## AI 의류 피팅 쇼핑 앱 적용 메모

- 앱 주제: AI 기반 의류 쇼핑 및 가상 피팅 앱
- 과제 조건 충족 방식:
  - 3개 이상의 Activity: 상품 목록, 상품 상세, AI 피팅, 결과 화면, 장바구니 등
  - API 및 네트워킹: AI virtual try-on API, 상품 API, 날씨 API 또는 자체 서버 통신
  - Kotlin: Android 앱 전체를 Kotlin으로 구현
- 독창성 포인트:
  - 단순 쇼핑 앱이 아니라 사용자의 사진과 선택한 의류 이미지를 이용해 AI가 착용 결과를 생성하는 기능 제공
- 주의할 점:
  - 영상과 보고서에는 실제 구현 및 데모 가능한 기능만 적는 것이 안전합니다.
  - AI API가 느리거나 실패할 수 있으므로 결과 화면, 로딩 화면, 실패 처리 등을 준비하는 것이 좋습니다.
